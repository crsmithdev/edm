---
description: Commit and push changes to GitHub
---

1. Run `git status` to see all changes
2. Run `git diff` to review staged and unstaged changes
3. Run `git log -3 --oneline` to see recent commit style
4. Draft a terse commit message following the repo's style (details in subject, body only if >100 chars)
5. Stage relevant files with `git add`
6. Commit with the message (include the standard footer)
7. Push to origin
8. Report the commit hash and push status
